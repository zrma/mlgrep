# v0.3 Multiple-File Memory Evidence

Status: pre-release v0.3.0 evidence

## Method

The v0.2 deterministic 128-byte line workload remains unchanged. The v0.3 gate
adds three native runs that process the 100 MiB fixture twice as two explicit
operands. This represents 200 MiB of sequential input while reusing no whole-file
buffer. `/usr/bin/time` supplies per-process peak RSS.

## Observation

On the recorded macOS arm64 run, the median 200 MiB aggregate workload completed
in about 2.46 seconds with about 1.36 MiB median peak RSS. The observed peak
growth across all 1, 10, 100 and aggregate 200 MiB workloads was about 0.03 MiB.
These values are observations, not portable performance guarantees.

## Gate

Single-file and sequential multiple-file measurements must each remain under
32 MiB peak RSS. Peak growth across every measured workload must remain under
16 MiB, and exact per-file count output is checked on every aggregate run.
